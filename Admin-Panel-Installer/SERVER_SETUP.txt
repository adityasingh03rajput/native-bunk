========================================
SERVER SETUP INFORMATION
For System Administrators
========================================

This document explains how to set up the backend
server that the Admin Panel connects to.

========================================

========================================

Before using the Admin Panel, ensure:

‚úì Backend server is installed and running
‚úì MongoDB database is running
‚úì Server is accessible on network
‚úì Firewall allows port 3000

========================================
üñ•Ô∏è SERVER REQUIREMENTS
========================================

The backend server needs:

- Node.js (v16 or higher)
- MongoDB (v4.4 or higher)
- Port 3000 available
- Network connectivity

========================================
üöÄ STARTING THE SERVER
========================================

On the server machine:

1. Navigate to server folder:
   cd path\to\project\server

2. Start MongoDB (if not running):
   mongod

3. Start the backend server:
   node index.js

4. Verify server is running:
   - Should show: "Server running on port 3000"
   - Should show: "Connected to MongoDB"

========================================
üåê FINDING YOUR SERVER URL
========================================

For Admin Panel configuration, you need the server URL:

Local Machine (same PC):
------------------------
http://localhost:3000

Same Network (LAN):
-------------------
1. On server PC, open Command Prompt
2. Type: ipconfig
3. Find "IPv4 Address" (e.g., 192.168.1.100)
4. Server URL: http://192.168.1.100:3000

Remote/Internet:
----------------
1. Use your public IP or domain name
2. Server URL: http://your-domain.com:3000
3. Ensure port 3000 is forwarded

========================================
üîß SERVER CONFIGURATION
========================================

Default Configuration:
- Port: 3000
- Database: MongoDB on localhost:27017
- Database Name: attendance_app

To change port, edit server/index.js:
const PORT = 3000; // Change this

========================================
üî• FIREWALL CONFIGURATION
========================================

Windows Firewall:
-----------------
1. Open Windows Defender Firewall
2. Advanced Settings
3. Inbound Rules ‚Üí New Rule
4. Port ‚Üí TCP ‚Üí 3000
5. Allow the connection
6. Apply to all profiles

Router Port Forwarding (for remote access):
-------------------------------------------
1. Access router admin panel
2. Port Forwarding section
3. Forward external port 3000 to server IP:3000
4. Save settings

========================================
‚úÖ TESTING SERVER CONNECTION
========================================

Test 1: Local Test
------------------
Open browser on server PC:
http://localhost:3000/api/health

Should show: {"status":"ok"}

Test 2: Network Test
--------------------
Open browser on another PC:
http://SERVER_IP:3000/api/health

Should show: {"status":"ok"}

Test 3: Admin Panel Test
-------------------------
1. Open Admin Panel
2. Go to Settings
3. Enter server URL
4. Check connection indicator (should be green)

========================================
üìä MONITORING SERVER
========================================

Check if server is running:

Windows:
--------
netstat -ano | findstr :3000

Should show active connection on port 3000

Check MongoDB:
--------------
mongo
show dbs
use attendance_app
show collections

Should show: students, teachers, timetables, etc.

========================================
üêõ TROUBLESHOOTING
========================================

Problem: Server won't start
----------------------------
‚Üí Check if port 3000 is already in use
‚Üí Verify MongoDB is running
‚Üí Check Node.js is installed: node --version

Problem: Cannot connect from Admin Panel
-----------------------------------------
‚Üí Verify server is running
‚Üí Check firewall settings
‚Üí Ping server IP from client PC
‚Üí Test URL in web browser first

Problem: Database connection failed
------------------------------------
‚Üí Start MongoDB: mongod
‚Üí Check MongoDB is running: mongo
‚Üí Verify database name in server config
